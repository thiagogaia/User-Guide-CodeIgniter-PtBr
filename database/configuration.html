Configuração do Banco de Dados¶

O CodeIgniter possui um arquivo de configuração que permite que você defina os dados de conexão do banco de dados (usuário, senha, nome do banco de dados, etc.). O arquivo de configuração está localizado no caminho: application/config/database.php. Você pode também definir valores de conexão do banco de dados para específicos ambientes colocando o arquivo database.php na pasta de configuração do respectivo ambiente.

As definições de configurações são definidas em um array multidimensional como neste exemplo: 

$db['default'] = array(
        'dsn'   => '',
        'hostname' => 'localhost',
        'username' => 'root',
        'password' => '',
        'database' => 'database_name',
        'dbdriver' => 'mysqli',
        'dbprefix' => '',
        'pconnect' => TRUE,
        'db_debug' => TRUE,
        'cache_on' => FALSE,
        'cachedir' => '',
        'char_set' => 'utf8',
        'dbcollat' => 'utf8_general_ci',
        'swap_pre' => '',
        'encrypt' => FALSE,
        'compress' => FALSE,
        'stricton' => FALSE,
        'failover' => array()
);

Alguns drivers de banco de dados (como por exemplo PDO, PostgreSQL, Oracle, ODBC) podem exigir o fornecimento do DNS completo. Se isso acontecer, você deve usar as definições de configuração ‘dsn’, como se você estivesse usando uma extensão de um driver PHP nativo subjacente, como esta: 

// PDO
$db['default']['dsn'] = 'pgsql:host=localhost;port=5432;dbname=database_name';

// Oracle
$db['default']['dsn'] = '//localhost/XE';

Nota

Se você não especificar o DNS para um driver que exige isso, o CodeIgniter irá tentar definir com o restante das configurações fornecidas.

Nota

Se você fornecer o DNS e estiver faltando alguma configuração válida (por exemplo, o conjunto de caracteres de banco de dados), que está presente no restante dos campos de configurações, o CodeIgniter irá acrescentá-los.

Você também pode especificar failovers para situações quando a principal conexão não consegue se conectar por alguma razão. Estes failovers podem ser especificados configurando o failover para uma conexão como esta:

$db['default']['failover'] = array(
                array(
                        'hostname' => 'localhost1',
                        'username' => '',
                        'password' => '',
                        'database' => '',
                        'dbdriver' => 'mysqli',
                        'dbprefix' => '',
                        'pconnect' => TRUE,
                        'db_debug' => TRUE,
                        'cache_on' => FALSE,
                        'cachedir' => '',
                        'char_set' => 'utf8',
                        'dbcollat' => 'utf8_general_ci',
                        'swap_pre' => '',
                        'encrypt' => FALSE,
                        'compress' => FALSE,
                        'stricton' => FALSE
                ),
                array(
                        'hostname' => 'localhost2',
                        'username' => '',
                        'password' => '',
                        'database' => '',
                        'dbdriver' => 'mysqli',
                        'dbprefix' => '',
                        'pconnect' => TRUE,
                        'db_debug' => TRUE,
                        'cache_on' => FALSE,
                        'cachedir' => '',
                        'char_set' => 'utf8',
                        'dbcollat' => 'utf8_general_ci',
                        'swap_pre' => '',
                        'encrypt' => FALSE,
                        'compress' => FALSE,
                        'stricton' => FALSE
                )
        );

Você pode especificar a quantidade de failovers que preferir.

A razão pelo qual usamos array multidimensionais ao invés de apenas um simples, é permitir que você possa definir múltiplos valores de conexões. Se, por exemplo, você rodar multiplos ambientes (desenvolvimento, produção, teste, etc.) dentro de uma única instalação, você pode configurar uma conexão para cada grupo, alternando-os como necessário. Por exemplo, para configurar um ambiente de “teste” você faria assim: 

$db['test'] = array(
        'dsn'   => '',
        'hostname' => 'localhost',
        'username' => 'root',
        'password' => '',
        'database' => 'database_name',
        'dbdriver' => 'mysqli',
        'dbprefix' => '',
        'pconnect' => TRUE,
        'db_debug' => TRUE,
        'cache_on' => FALSE,
        'cachedir' => '',
        'char_set' => 'utf8',
        'dbcollat' => 'utf8_general_ci',
        'swap_pre' => '',
        'compress' => FALSE,
        'encrypt' => FALSE,
        'stricton' => FALSE,
        'failover' => array()
);

Então, para dizer globalmente ao sistema para usar esse grupo você deve definir esta variável localizada no arquivo de configuração:

$active_group = 'test';

Nota

O nome ‘test’ é arbitrário e pode ser o nome que você preferir. Por padrão estamos usando a palavra “default” para a conexão primária, mas que também pode ser renomeado para algum nome que possa ser mais relevante para o seu projeto.

Construção de Consultas¶

A Classe de Contrução de Consulta é globalmente habilitada ou desabilitada definindo a variável $query_builder no arquivo de configuração do banco de dados para TRUE/FALSE (boolean). A configuração padrão é TRUE. Se você não estiver utilizando a classe de construção de consultas, defina $query_builder como FALSE, desta forma irá utilizar menos recursos quando a classe de banco de dados for inicializada.

$query_builder = TRUE;

Nota

Algumas classes do CodeIgniter como por exemplo as classes de Sessões requerem que a Construção de Consultas esteja habilitada para acessar determinadas funcionalidades.

Definição de Valoes:¶

Nome da Configuração Descrição
dsn Configuração de acesso ao banco de dados em apenas uma linha.
hostname O nome do host do seu servidor de banco de dados. Geralmente é ‘localhost’.
username O nome de usuário usada para conectar ao banco de dados.
password Senha usada para conectar ao banco de dados.
database O nome do banco de dados que você deseja se conectar.
dbdriver O tipo de banco de dados. Por exemplo: mysqli, postgre, odbc, etc. Deve ser especificado em letra minúscula.
dbprefix Um prefixo de tabela opcional que será adicionado ao nome da tabela quando houver a execução de consultas no Construtor de Consultas. Isso permite que múltiplas instalações CodeIgniter compartilhem um banco de dados
pconnect TRUE/FALSE (boolean) - Se for utilizar uma conexão persistente.
db_debug TRUE/FALSE (boolean) - Ativar ou desativar a exibição dos erros de banco de dados na tela.
cache_on TRUE/FALSE (boolean) - Se o cache de consulta do banco de dados está habilitado, veja também Classe de Cacheamento de Banco de Dados.
cachedir Caminho absoluto do servidor para o diretório de cache de consultas do seu banco de dados.
char_set O conjunto de caracteres usado na comunicação com o banco de dados.
dbcollat

A coleção de caracteres usados na comunicação com o banco de dados

Nota

Somente usado nos drivers ‘mysql’ e ‘mysqli’.
swap_pre O prefixo de tabela padrão que deve ser trocado com dbprefix. É muito útil para aplicações distribuídas onde você deve rodar consultas escritas manualmente, e precisa que o prefixo ainda esteja customizável pelo usuário final.
schema É o esquema de banco de dados, que por padrão é ‘public’. Usado pelos drivers de banco de dados PostgreSQL e ODBC.
encrypt

Se usa ou não uma conexão criptografada.

  • Drivers ‘mysql’ (obsoleto), ‘sqlsrv’ e ‘pdo/sqlsrv’ aceitam TRUE/FALSE
  • Drivers ‘mysqli’ e ‘pdo/mysql’ aceitam um array com as seguintes opções:
    • ‘ssl_key’ - Caminho para o arquivo com a chave privada
    • ‘ssl_cert’ - Caminho para o arquivo com a chave pública certificada
    • ‘ssl_ca’ - Caminho para o arquivo com o certificado autorizado
    • ‘ssl_capath’ - Caminho para o diretório contendo certificados CA confiáveis no formato PEM
    • ‘ssl_cipher’ - Lista de cifras permitidas para serem usadas para a criptografia, separadas por dois pontos (‘:’)
    • ‘ssl_verify’ - TRUE/FALSE; Se verificar (somente ‘mysqli’)
    • ‘ssl_verify’ - TRUE/FALSE; Para verificar o certificado do servidor ou não (somente ‘mysqli’)
compress Se deve ou não utilizar a compressão de cliente (somente MySQL).
stricton TRUE/FALSE (boolean) - Forçar conexões “no modo estrito”, bom para garantir SQL estrito durante o desenvolvimento de uma aplicação.
port

O número da porta do banco de dados. Para usar este valor você tem que adicionar uma linha no array de configuração do banco de dados.

$db['default']['port'] = 5432;

Nota

Dependendo de qual plataforma de banco de dados você está usando (MySQL, PostgreSQL, etc.) não será necessário todos os valores. Por exemplo, quando se utiliza o SQLite você não precisará fornecer um nome de usuário ou senha, bem como o nome do banco de dados será o caminho para o arquivo de banco de dados. As informações acima assume que você esteja usando o MySQL.
