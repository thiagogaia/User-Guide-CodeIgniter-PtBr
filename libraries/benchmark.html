Classe de Benchmark¶

CodeIgniter tem uma classe de Benchmarking que está sempre ativa, permitindo que a diferença horária entre os dois pontos marcados seja calculada.

Nota

Esta classe é inicializada automaticamente pelo sistema, portanto, não há necessidade de fazê-lo manualmente.

Além disso, o benchmark sempre é iniciado no momento em que a estrutura é invocada e terminada pela classe de saída logo antes de enviar a visão final para o navegador, permitindo que seja exibido um tempo muito preciso de execução total do sistema.

Usando a Classe de Benchmark¶

A classe Benchmark pode ser usada dentro de seus controllers, visões, ou em seus models. O processo de uso é o seguinte:

  1. Marque um ponto de partida
  2. Marque um ponto final
  3. Execute a função "tempo decorrido" para ver os resultados

Aqui está um exemplo usando o código real:

$this->benchmark->mark('code_start');

// Algum código acontece aqui

$this->benchmark->mark('code_end');

echo $this->benchmark->elapsed_time('code_start', 'code_end');

Nota

As palavras "code_start" e "code_end" são arbitrárias. São apenas palavras usadas para definir dois marcadores. Você pode usar as palavras que deseja, e você pode definir vários conjuntos de marcadores. Considere este exemplo:

$this->benchmark->mark('dog');

// Algum código acontece aqui

$this->benchmark->mark('cat');

// Mais algum código acontece aqui

$this->benchmark->mark('bird');

echo $this->benchmark->elapsed_time('dog', 'cat');
echo $this->benchmark->elapsed_time('cat', 'bird');
echo $this->benchmark->elapsed_time('dog', 'bird');

Perfis dos Pontos de Benchmarking¶

Se você quiser que seus dados de referência estejam disponíveis para Perfis, todos os seus pontos marcados devem ser configurados em pares e cada nome do ponto de marca deve terminar com _start e _end. Cada par de pontos devem ser nomeados de forma idêntica. Exemplo:

$this->benchmark->mark('my_mark_start');

// Some code happens here...

$this->benchmark->mark('my_mark_end');

$this->benchmark->mark('another_mark_start');

// Some more code happens here...

$this->benchmark->mark('another_mark_end');

Leia a página Perfis para obter mais informações.

Mostrando o Tempo Total de Execução¶

Se você quiser exibir o tempo total decorrido a partir do momento em que o CodeIgniter começa até o momento em que a saída final é enviada para o navegador, basta colocar isso em um dos suas visões de exibição:

<?php echo $this->benchmark->elapsed_time();?>

Você notará que é a mesma função usada nos exemplos acima para calcular o tempo entre dois pontos, exceto que você não está usando nenhum parâmetro. Quando os parâmetros estão ausentes, o CodeIgniter não para o benchmark até logo antes que o resultado final seja enviado ao navegador. Não importa onde você use a chamada de função, o temporizador continuará a funcionar até o final.

Uma maneira alternativa de mostrar o tempo decorrido em seus arquivos de exibição é usar essa pseudo-variável, se você preferir não usar o PHP puro:

{elapsed_time}

Nota

Se você quiser comparar tudo nas funções do seu controlador, você deve definir seus próprios pontos de início/fim.

Mostrando o Consumo de Memória¶

Se sua instalação PHP estiver configurada com -enable-memory-limit, você pode exibir a quantidade de memória consumida por todo o sistema usando o seguinte código em um de seus arquivos de exibição:

<?php echo $this->benchmark->memory_usage();?>

Nota

Esta função só pode ser usada em seus arquivos de exibição. O consumo refletirá a memória total usada por todo o aplicativo.

Uma maneira alternativa de mostrar seu uso de memória em seus arquivos de exibição é usar esta pseudo-variável, se você preferir não usar o PHP puro:

{memory_usage}

Referência de Classe¶

class CI_Benchmark¶
mark($name)¶
Parâmetros:
  • $name (string) – O nome que você deseja atribuir ao seu marcador.
Tipo de retorno:

nada

Define um marcador de referência..

elapsed_time([$point1 = ''[, $point2 = ''[, $decimals = 4]]])¶
Parâmetros:
  • $point1 (string) – a particular marked point
  • $point2 (string) – a particular marked point
  • $decimals (int) – number of decimal places for precision
Retorna:

Tempo gasto
Tipo de retorno:

string

Calcula e retorna a diferença de tempo entre dois pontos marcados.

Se o primeiro parâmetro estiver vazio, esta função retornará o {elapsed_time} Pseudo-variável. Isso permite que o tempo total de execução do sistema seja mostrado em um modelo. A classe de saída irá trocar o valor real dessa variável

memory_usage()¶
Retorna:Informações de uso da memória
Tipo de retorno:string

Simply returns the {memory_usage} marcador.

Isso permite que ele seja colocado em qualquer lugar em um modelo sem a memória sendo calculada até o fim. A Classe de Saída will irá trocar o valor real dessa variável.
