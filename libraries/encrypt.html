Classe de Criptografia¶

A Classe de Criptografia fornece criptografia de dados bidirecional. Ele criptografou usando a extensão Mcrypt PHP, que é necessária para que a Classe de Criptografia seja executada.

Importante

Esta biblioteca foi DEPRECIADA e é mantida somente para compatibilidade com versões anteriores. Use a nova Biblioteca de Criptografia.

Usando a Biblioteca de Criptografia¶

Configurando sua chave¶

Uma chave é uma informação que controla o processo criptográfico e permite que uma seqüência criptografada seja decodificada. Na verdade, a chave que você escolherá fornecerá o único meio para decodificar dados que foram criptografados com essa chave, portanto, não só você deve escolher a chave com cuidado, você nunca deve alterá-la se você pretende usá-la para dados persistentes.

Sem dizer que você deve proteger sua chave com cuidado. Se alguém tiver acesso à sua chave, os dados serão facilmente decodificados. Se o seu servidor não estiver totalmente sob seu controle, é impossível garantir a segurança das chaves, então você pode querer pensar com cuidado antes de usá-lo para qualquer coisa que exija alta segurança, como armazenar números de cartão de crédito.

Para aproveitar ao máximo o algoritmo de criptografia, sua chave deve ter 32 caracteres de comprimento (256 bits). A chave deve ser uma seqüência aleatória como você pode inventar, com números e letras maiúsculas e minúsculas. Sua chave não deve ser uma string de texto simples. Para ser criptográficamente seguro, ele precisa ser o mais aleatório possível.

Sua chave pode ser armazenada em application/config/config.php, ou você pode projetar seu próprio mecanismo de armazenamento e passar a chave dinamicamente ao codificar/decodificar.

Para salvar sua chave em application/config/config.php, abra o arquivo e defina:

$config['encryption_key'] = "YOUR KEY";

Tamanho da mensagem¶

É importante que você saiba que as mensagens codificadas geradas pela função de criptografia serão aproximadamente 2,6 vezes mais do que a mensagem original. Por exemplo, se você criptografar a string "my super secrets", que tem 21 caracteres, você acabará com uma string codificada com aproximadamente 55 caracteres (dizemos "grosseiramente" porque o comprimento da seqüência codificada aumenta em 64 Bit clusters, então não é exatamente linear). Tenha em mente esta informação ao selecionar seu mecanismo de armazenamento de dados. Cookies, por exemplo, só podem conter 4K de informações.

Inicializando a Classe¶

Como a maioria das outras classes no CodeIgniter, a classe Encrypt é inicializada no seu controlador usando o método $this->load->library():

$this->load->library('encrypt');

Uma vez carregado, o objeto da biblioteca Encrypt estará disponível usando:

$this->encrypt

Referência de Classe¶

class CI_Encrypt¶
encode($string[, $key = ''])¶
Parâmetros:
  • $string (string) – Dados para criptografar
  • $key (string) – Chave de criptografia
Retorna:

String Criptografada
Tipo de retorno:

string

Executa a criptografia de dados e retorna como uma string. Exemplo:

$msg = 'My secret message';

$encrypted_string = $this->encrypt->encode($msg);

Você pode, opcionalmente, passar a sua chave de criptografia através do segundo parâmetro, se você não quiser usar o do seu arquivo de configuração:

$msg = 'My secret message';
$key = 'super-secret-key';

$encrypted_string = $this->encrypt->encode($msg, $key);
decode($string[, $key = ''])¶
Parâmetros:
  • $string (string) – String para descriptografar
  • $key (string) – Chave Criptografada
Retorna:

String de Texto Plano
Tipo de retorno:

string

Decodifica uma string codificada. Exemplo:

$encrypted_string = 'APANtByIGI1BpVXZTJgcsAG8GZl8pdwwa84';

$plaintext_string = $this->encrypt->decode($encrypted_string);

Você pode, opcionalmente, passar a sua chave de criptografia através do segundo parâmetro, se você não quiser usar o do seu arquivo de configuração:

$msg = 'My secret message';
$key = 'super-secret-key';

$encrypted_string = $this->encrypt->decode($msg, $key);
set_cipher($cipher)¶
Parâmetros:
  • $cipher (int) – Constante Criptografada Válida do PHP MCrypt
Retorna:

Instância do CI_Encrypt (método de encadeamento)
Tipo de retorno:

CI_Encrypt

Permite que você defina uma criptografia Mcrypt. Por padrão, ele usa MCRYPT_RIJNDAEL_256. Exemplo:

$this->encrypt->set_cipher(MCRYPT_BLOWFISH);

Visite php.net para obter uma lista de criptografias disponíveis.

Se você quiser verificar manualmente se o seu servidor aceita o MCrypt, você pode usar:

echo extension_loaded('mcrypt') ? 'Yup' : 'Nope';
set_mode($mode)¶
Parâmetros:
  • $mode (int) – Constante válida do modo MCrypt PHP
Retorna:

Instância do CI_Encrypt (método de encadeamento)
Tipo de retorno:

CI_Encrypt

Permite que você defina um modo Mcrypt. Por padrão, ele usa MCRYPT_MODE_CBC. Exemplo:

$this->encrypt->set_mode(MCRYPT_MODE_CFB);

Visite php.net para obter uma lista de criptografias disponíveis.

encode_from_legacy($string[, $legacy_mode = MCRYPT_MODE_ECB[, $key = '']])¶
Parâmetros:
  • $string (string) – String para criptografar
  • $legacy_mode (int) – Constante válida do modo MCrypt PHP
  • $key (string) – Chave de criptografia
Retorna:

String recente criptografada
Tipo de retorno:

string

Permite recodificar os dados originalmente criptografados com CodeIgniter 1.x para serem compatíveis com a biblioteca Encrypt no CodeIgniter 2.x. Só é necessário usar este método se você tiver dados criptografados armazenados permanentemente, como em um arquivo ou banco de dados e estiverem em um servidor que suporte o Mcrypt. A criptografia de uso "Light", como dados de sessão criptografados ou flashdata criptografado transitório, não requer intervenção de sua parte. No entanto, as sessões criptografadas existentes serão destruídas uma vez que os dados criptografados antes de 2.x não serão decodificados.

Importante

Por que apenas um método para codificar novamente os dados em vez de manter métodos legados para codificação e decodificação? The algorithms in the Os algoritmos na biblioteca Encrypt melhoraram no CodeIgniter 2.x tanto para desempenho quanto para segurança, e não desejamos encorajar o uso contínuo dos métodos mais antigos. Você pode, naturalmente, estender a biblioteca de Criptografia se desejar e substituir os novos métodos com o antigo e manter a compatibilidade perfeita com os dados criptografados CodeIgniter 1.x, mas essa é uma decisão que um desenvolvedor deve fazer de forma cautelosa e deliberada, se for do todo.

$new_data = $this->encrypt->encode_from_legacy($old_encrypted_string);
Parâmetros Padrão Descrição
$orig_data n/a Os dados criptografados originais da biblioteca de criptografia CodeIgniter 1.x
$legacy_mode MCRYPT_MODE_ECB O modo Mcrypt que foi usado para gerar os dados criptografados originais. O padrão do CodeIgniter 1.x foi MCRYPT_MODE_ECB, e assumirá que, para ser o caso, a menos que seja substituído por este parâmetro.
$key n/a A chave de criptografia. Isso geralmente é especificado em seu arquivo de configuração conforme descrito acima.
