Funções Comuns¶

CodeIgniter usa algumas funções para o seu funcionamento definidas globalmente e estão disponíveis para você em qualquer ponto. Estes não exigem o carregamento de bibliotecas ou auxiliares.

is_php($version)¶
Parâmetros:
  • $version (string) – Número da versão
Retorna:

VERDADEIRO Se a versão PHP em execução for pelo menos a especificada ou FALSO se não for.
Tipo de retorno:

booleano

Determina se a versão do PHP que está sendo usada é maior do que o número da versão fornecida.

Exemplo:

if (is_php('5.3'))
{
        $str = quoted_printable_encode($str);
}

Retorna TRUE se a versão instalada do PHP for igual ou superior ao número de versão fornecido. Retorna FALSO se a versão instalada do PHP for menor do que o número de versão fornecido.

is_really_writable($file)¶
Parâmetros:
  • $file (string) – Caminho do arquivo
Retorna:

VERDADEIRO se o caminho for gravável.
Tipo de retorno:

booleano

is_writable() retorna VERDADEIRO nos servidores Windows quando você realmente não pode gravar no arquivo, pois o sistema operacional reporta ao PHP como FALSO somente se o atributo somente leitura estiver marcado.

Esta função determina se um arquivo realmente pode ser gravado, tentando gravar primeiro. Geralmente, apenas é recomendado em plataformas onde esta informação pode não ser confiável.

Exemplo:

if (is_really_writable('file.txt'))
{
        echo "Eu posso gravar nesse arquivo se eu quiser";
}
else
{
        echo "O arquivo não pode ser gravado";
}

Nota

Veja também PHP bug #54709 para mais informações.

config_item($key)¶
Parâmetros:
  • $key (string) – Chave do item
Retorna:

O valor da chave de configuração, ou NULO se não encontrado.
Tipo de retorno:

Vários tipos

A Biblioteca de configuração é a maneira preferida de acessar informações de configuração, no entanto config_item() pode ser usado para recuperar chaves. Veja a documentação da Biblioteca de configuração para mais informações.

set_status_header($code[, $text = ''])¶
Parâmetros:
  • $code (int) – HTTP Código de status de resposta HTTP
  • $text (string) – Uma mensagem personalizada para definir com o código de status
Tipo de retorno:

NADA

Permite que você defina manualmente um cabeçalho de status do servidor. Exemplo:

set_status_header(401);
// Define o cabeçalho como: Não autorizado

Veja aqui para consultar a lista completa de cabeçalhos

remove_invisible_characters($str[, $url_encoded = TRUE])¶
Parâmetros:
  • $str (string) – Conjunto de caracteres de entrada
  • $url_encoded (bool) – Seja para remover caracteres codificados em URL também
Retorna:

Conjunto de caracteres sanitizados
Tipo de retorno:

Conjunto de caracteres

Esta função impede a inserção de caracteres NULL entre caracteres ASCII, como Java \ 0script.

Exemplo:

remove_invisible_characters('Java\\0script');
// Retorna: 'Javascript'
html_escape($var)¶
Parâmetros:
  • $var (mixed) – Variável para escapar (sequência de caracteres ou matriz)
Retorna:

HTML com sequência de caractere(s) escapado(s)
Tipo de retorno:

Vários tipos

Esta função funciona como um alias para a função nativa do PHP htmlspecialchars() com a vantagem de ser capaz de aceitar uma matriz com sequência de caracteres.

É útil na prevenção do Cross Site Scripting (XSS).

get_mimes()¶
Retorna:Uma matriz associativa de tipos de arquivos
Tipo de retorno:matriz

Esta função retorna uma referência à matriz de MIME em application/config/mimes.php.

is_https()¶
Retorna:VERDADEIRO se estiver usando HTTP-over-SSL, FALSO se não
Tipo de retorno:booleano

Retorna TRUE se uma conexão segura (HTTPS) for usada e FALSO em qualquer outro caso (incluindo solicitações não HTTP).

is_cli()¶
Retorna:VERDADEIRO se estiver executando atualmente em CLI, FALSO caso contrário
Tipo de retorno:booleano

Retorna TRUE se o aplicativo for executado através da linha de comando e FALSE se não.

Nota

Esta função verifica se o valor de PHP_SAPI é ‘cli’ ou se a constante STDIN foi definida.

function_usable($function_name)¶
Parâmetros:
  • $function_name (string) – Nome da função
Retorna:

VERDADEIRO se a função puder ser usada, FALSO se não
Tipo de retorno:

booleano

Retorna VERDADEIRO se uma função existe e é utilizável, ou FALSO caso contrário

Esta função executa uma verificação function_exists() e se a extensão Suhosin <http://www.hardened-php.net/suhosin/> está carregada, verifica se não desabilita a função que está sendo verificada.

É útil se você deseja verificar a disponibilidade de funções como eval() e exec() , que são perigosas e podem ser desativadas em servidores com políticas de segurança altamente restritivas.

Nota

Esta função foi introduzida porque Suhosin terminou a execução do script, mas isso acabou por ser um bug. Uma correção está disponível há algum tempo (versão 0.9.34), mas infelizmente ainda não foi lançado.
