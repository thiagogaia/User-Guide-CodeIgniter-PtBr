Criando Bibliotecas¶

Quando usamos o termo “Biblioteca” Normalmente estamos nos referindo às classes que estão localizadas no diretório das bibliotecas e descritas na Referência de Classe deste guia do usuário. Neste caso, no entanto, descreveremos como você pode criar suas próprias bibliotecas dentro do seu diretório application/libraries para manter a separação entre seus recursos locais e os recursos da estrutura global.

Como um bônus adicional, o CodeIgniter permite que suas bibliotecas ampliem as classes nativas se você simplesmente precisar adicionar algumas funcionalidades a uma biblioteca existente. Ou você pode até mesmo substituir bibliotecas nativas apenas colocando versões de nome idêntico em seu diretório application/libraries.

Em resumo:

  • Você pode criar bibliotecas inteiramente novas.
  • Você pode ampliar as bibliotecas nativas.
  • Você pode substituir bibliotecas nativas.

A página abaixo explica esses três conceitos em detalhes..

Nota

As classes de banco de dados não podem ser estendidas ou substituídas por suas próprias classes. Todas as outras classes podem ser substituídas/estendidas.

Armazenamento¶

Suas classes de biblioteca devem ser inseridas dentro do diretório application/libraries , pois é aqui que o CodeIgniter irá procurá-las quando forem inicializadas.

Convenções de nomenclatura¶

  • Os nomes dos arquivos devem começar com letras maiúsculas. Por exemplo: Myclass.php
  • As declarações de classe devem ser capitalizadas. Por exemplo: classe Myclass
  • Nomes de classe e nomes de arquivos devem ser iguais

O Arquivo de Classe¶

Classes devem seguir esse padrão:

<?php
defined('BASEPATH') OR exit('No direct script access allowed');

class Someclass {

        public function some_method()
        {
        }
}

Nota

Estamos usando o nome Someclass como um exemplo

Usando sua Classe¶

De dentro de qualquer um dos seus métodos Controladores você pode inicializar sua classe usando o padrão:

$this->load->library('someclass');

Onde someclass é o nome do arquivo, sem a extensão ”.php”. Você pode enviar o nome do arquivo em maiúsculas ou minúsculas. O CodeIgniter não se importa.

Uma vez carregado, você pode acessar sua classe usando a versão em minúsculas:

$this->someclass->some_method();  // As instâncias de objeto sempre serão minúsculas

Passando Parâmetros ao Inicializar sua Classe¶

No método de carregamento da biblioteca, você pode passar dinamicamente dados como uma matriz através do segundo parâmetro e será passado para seu construtor de classe:

$params = array('type' => 'large', 'color' => 'red');

$this->load->library('someclass', $params);

Se você usar esse recurso, você deve configurar o construtor da sua classe para esperar dados:

<?php defined('BASEPATH') OR exit('No direct script access allowed');

class Someclass {

        public function __construct($params)
        {
                // Do something with $params
        }
}

Você também pode passar parâmetros armazenados em um arquivo de configuração. Basta criar um arquivo de configuração denominado de forma idêntica ao nome do arquivo de classe e armazená-lo no diretório application/config/. Observe que, se você passar dinamicamente parâmetros conforme descrito acima, a opção do arquivo de configuração não estará disponível.

Utilizando Recursos do Codeigniter com sua Biblioteca¶

Para acessar os recursos nativos do CodeIgniter dentro de sua biblioteca use o método get_instance() Esse método retorna o super objeto do CodeIgniter.

Normalmente, dentro dos seus métodos de controle, você chamará qualquer um dos métodos CodeIgniter disponíveis usando o construtor $this :

$this->load->helper('url');
$this->load->library('session');
$this->config->item('base_url');
// etc.

$this, no entanto, só funciona diretamente em seus controladores, seus modelos ou suas visualizações. Se você quiser usar as classes do CodeIgniter a partir de suas próprias classes personalizadas, você pode fazê-lo da seguinte maneira::

Primeiro, instancie o objeto CodeIgniter a uma variável:

$CI =& get_instance();

Depois de atribuir o objeto a uma variável, você usará essa variável ao invés de $this:

$CI =& get_instance();

$CI->load->helper('url');
$CI->load->library('session');
$CI->config->item('base_url');
// etc.

Nota

Você notará que na função acima get_instance() está sendo passada pela referência:

$CI =& get_instance();

Isto é muito importante. Atribuir por referência permite que você use o objeto CodeIgniter original em vez de criar uma cópia dele.

No entanto, uma vez que uma biblioteca é uma classe, seria melhor se você aproveitar ao máximo os princípios OOP. Então, para poder usar o super-objeto CodeIgniter em todos os métodos de classe, você é encorajado a atribuí-lo a uma propriedade em vez disso:

class Example_library {

        protected $CI;

        // Usaremos um construtor, pois você não pode chamar diretamente uma função
        // de uma definição de propriedade.
        public function __construct()
        {
                // Atribuir o Super objeto CodeIgniter
                $this->CI =& get_instance();
        }

        public function foo()
        {
                $this->CI->load->helper('url');
                redirect();
        }

        public function bar()
        {
                echo $this->CI->config->item('base_url');
        }

}

Substituindo Bibliotecas Nativas por suas Versões¶

Basta nomear os arquivos da sua classe de forma idêntica a uma biblioteca nativa fará com que CodeIgniter o use em vez do nativo. Para usar esse recurso, você deve nomear o arquivo e a declaração de classe exatamente o mesmo que a biblioteca nativa. Por exemplo, para substituir a biblioteca nativa de e-mail, você criará um arquivo chamado application/libraries/Email.php, e declarará sua classe com:

class CI_Email {

}

Observe que a maioria das classes nativas são prefixadas com CI_.

Para carregar sua biblioteca, você verá o método de carregamento padrão:

$this->load->library('email');

Nota

Neste momento, as classes de banco de dados não podem ser substituídas por suas próprias versões.

Estendendo Bibliotecas Nativas¶

Se tudo o que você precisa fazer é adicionar algumas funcionalidades a uma biblioteca existente - talvez adicione um método ou dois - então é um excesso para substituir toda a biblioteca pela sua versão. Neste caso, é melhor simplesmente estender a classe. A extensão de uma classe é quase idêntica à substituição de uma classe por algumas exceções:

  • A declaração de classe deve estender a classe pai.
  • Seu novo nome de classe e seu nome de arquivo devem ser prefixados com MY_ (este item é configurável. Veja abaixo).

Por exemplo, para estender a classe nativa de e-mail, você criará um arquivo chamado application/libraries/MY_Email.php, e declare sua classe com:

class MY_Email extends CI_Email {

}

Se você precisar usar um construtor em sua classe, certifique-se de estender o construtor pai:

class MY_Email extends CI_Email {

        public function __construct($config = array())
        {
                parent::__construct($config);
        }

}

Note

Nem todas as bibliotecas têm os mesmos (ou quaisquer) parâmetros em seu construtor. Dê uma olhada na biblioteca que você está expandindo primeiro para ver como ela deve ser implementada.

Carregando sua Sub-classe¶

Para carregar sua subclasse, você usará a sintaxe padrão normalmente usada. NÃO inclua seu prefixo. Por exemplo, para carregar o exemplo acima, que estende a classe Email, você usará:

$this->load->library('email');

Uma vez carregado, você usará a variável de classe como você normalmente faria para a classe que você está estendendo. No caso da classe de e-mail, todas as chamadas usarão:

$this->email->some_method();

Configurando seu Próprio Prefixo¶

Para definir seu próprio prefixo de sub-classe, abra o arquivo application/config/config.php e olhe este item:

$config['subclass_prefix'] = 'MY_';

Observe que todas as bibliotecas do CodeIgniter nativas são prefixadas com CI_ então NÃO use isso como seu prefixo.
